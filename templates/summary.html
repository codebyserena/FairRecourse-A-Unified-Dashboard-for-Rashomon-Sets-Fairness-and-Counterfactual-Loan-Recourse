Project Overview

Project Summary

A practical dashboard for credit-risk modeling, fairness checks, explainability, and recourse.

What This Project Does

FairRecourse is a machine learning dashboard for analyzing credit-risk decisions. It trains multiple models, compares their accuracy, checks fairness across sensitive groups, explains predictions, and generates counterfactual suggestions for applicants who are predicted to be rejected.

The project is built as a complete workflow: data cleaning, preprocessing, model training, fairness auditing, explainability, recourse generation, and an interactive Flask dashboard for presenting the results.

1

Model Comparison

The pipeline trains several model families, including Logistic Regression, Decision Trees, Random Forests, and Gradient Boosting. The dashboard compares their accuracy and highlights models that perform close to the best result.

2

Fairness & Explainability

Fairlearn is used to calculate fairness gaps across sex, education, and age bands. SHAP is used to show which features contribute most to a model prediction, making the model output easier to interpret.

3

Recourse Suggestions

DiCE is used to generate counterfactual examples. These suggestions show which feature values would need to change for a rejected applicant to move toward approval under different models.

Why It Is Useful

This project is useful because it brings together the parts of a responsible ML workflow that are often checked separately: model performance, fairness, explainability, and applicant-level recourse.

Instead of only showing which model is most accurate, the dashboard helps compare the tradeoffs between accuracy, fairness, and actionability in a clear visual format.
